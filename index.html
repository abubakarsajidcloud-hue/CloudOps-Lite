04 · Infrastructure as Code

Terraform Modules

Nine HCL modules describe every resource — fully declarative, version-controlled and reproducible. Standard workflow: init → validate → plan → apply.

variables.tfReusable input variables
vpc.tfVPC, subnet, IGW, routes
ec2.tfInstance + security group
s3.tfStorage bucket
iam.tfRole + policy attachment
cloudwatch.tfCPU metric alarm
sns.tfAlert topic
outputs.tfPublic IP, VPC ID, bucket
05 · Automation

Bash Operations

Six scripts wrap the entire operational lifecycle — deploy, observe, cost-report, optimize, report and destroy.

deploy.sh
provision

Runs terraform fmt, validate, plan and apply in sequence to provision the entire stack.

./deploy.sh
monitor.sh
observe

Pulls CloudWatch CPU metrics through the AWS CLI and renders a health report on the terminal.

./monitor.sh
cost-report.sh
finops

Queries AWS Cost Explorer for MTD spend, per-service breakdown and a month-end forecast.

./cost-report.sh
shutdown-idle.sh
optimize

Detects EC2 instances with CPU < 5% for 2 hours and stops them automatically to reduce cost.

./shutdown-idle.sh
report.sh
report

Aggregates infrastructure state and writes operational reports into the reports/ directory.

./report.sh
destroy.sh
teardown

Safely tears down all provisioned AWS resources via terraform destroy.

./destroy.sh
06 · Deployment

From Zero to Production

Six commands to take the platform from a clean machine to a fully operational, observed and cost-tracked AWS environment.

Configure AWS CLI aws configure

Provide access key, secret, region and default output format.

Initialize Terraform terraform init

Download providers and prepare the working directory.

Deploy Infrastructure ./deploy.sh

Provision VPC, EC2, S3, IAM, CloudWatch and SNS in one command.

Monitor Health ./monitor.sh

Stream CloudWatch metrics directly to the console.

Track Cost ./cost-report.sh

Generate cost breakdown and a month-end forecast.

Optimize & Report ./shutdown-idle.sh

Auto-stop idle instances and archive operational reports.
